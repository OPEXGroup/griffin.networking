Interfaces

  InterfaceDescription
IServerClientContext
Context used by application clients to be able to send stuff.
IServerService
Represents that class that will handle a connection from the client
IServiceFactory
Used to create the client (i.e. your class that handles a connection from a client)

Remarks

All servers are divided into two parts. The core which will accept all clients and monitor their state (all classes derived from ServerBase) and the actual class which takes care of each client (IServerService).
